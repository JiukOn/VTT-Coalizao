================================================================================
  1. TEST LOG STRUCTURE
================================================================================

Each test performed must be logged as a file in the folder:
  docs/Logs/

File naming convention:
  YYYY-MM-DD_PhaseX_TestName.txt

Example:
  2026-04-10_Phase1_DesignSystemTest.txt
  2026-04-12_Phase2_DiceRollerTest.txt


================================================================================
  2. TEST LOG TEMPLATE
================================================================================

Copy the template below for each log file:

[Briefly describe what is being validated]

================================================================================
  4. BUG SEVERITY
================================================================================

  🔴 CRITICAL — App breaks, data loss, fails to load
  🟠 HIGH     — Core functionality does not work
  🟡 MEDIUM   — Visible bug but with a workaround
  🟢 LOW      — Cosmetic, does not affect functionality


================================================================================
  5. NAMING FORMAT FOR BUGS
================================================================================

  [BUG-PXXX] where P = Phase and XXX = sequential number
  Examples: BUG-1001, BUG-2003, BUG-4012
