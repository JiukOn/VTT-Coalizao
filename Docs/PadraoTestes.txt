================================================================================
  1. ESTRUTURA DO LOG DE TESTE
================================================================================

Cada teste realizado deve ser registrado como um arquivo na pasta:
  docs/Logs/

Nomeação do arquivo:
  YYYY-MM-DD_FaseX_NomeDoTeste.txt

Exemplo:
  2026-04-10_Fase1_TesteDesignSystem.txt
  2026-04-12_Fase2_TesteDiceRoller.txt


================================================================================

================================================================================
  4. SEVERIDADE DE BUGS
================================================================================

  🔴 CRÍTICA — App quebra, perde dados, não carrega
  🟠 ALTA    — Funcionalidade principal não funciona
  🟡 MÉDIA   — Bug visível mas com workaround
  🟢 BAIXA   — Cosmético, não afeta funcionalidade

  5. FORMATO DE NOME PARA BUGS
================================================================================

  [BUG-FXXX] onde F = Fase e XXX = número sequencial
  Exemplos: BUG-1001, BUG-2003, BUG-4012
